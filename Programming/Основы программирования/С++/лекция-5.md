# Пространства имен и перегрузка функций

## Псевдонимы 
### Reference (lvalue reference)
Ссылка. Ссылка является псевдонимом объекта. Обязательно должна быть инициализирована.

```cpp
#include <iostream>

int main() {
	int x = 10;
	int &y = x;
	std::cout << x << " " << y << '\n'; // "10 10"
	y = 25;
	std::cout << x << " " << y << '\n'; // "25 25"
	x = 23;
	std::cout << x << " " << y << '\n'; // "23 23"
	return 0;
}
```

UB:
```cpp
#include <iostream>

int& foo() {
	int i = 10;
	return i;
}

int main() {
	int& i = foo();
	std::cout << i; // undefinded behaivour
	return 0;
}
```

Поскольку ссылка определена на стеке, то после завершения `foo()` место освободится, а значит ссылка никуда не ведет и получается UB.
### Reference (rvalue reference)
Ссылка для передачи владения объектом. Rlavue ссылка может быть только одна.

```cpp
#include <iostream>

int main() {
	int&& rv = 10
	return 0;
}
```


## Перегрузка функций

Несколько функций имеют одинаковые названия, НО разные аргументы. При этом  передаваемые аргументы в функцию должны быть явно указаны, иначе компилятор не будет знать к какому типу преобразовывать переменную.
