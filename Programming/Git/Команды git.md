- `git init` - инициализирует новый локальный репозиторий **git**.
		Создает скрытую папку `.git` в текущей директории, где хранится вся история изменений, ветки, настройки и другие служебные данные. Выполняется один раз для начала работы с **git** в проекте.
- `git status` - показывает текущее состояние репозитория.
		Отображает измененные файлы, готовые к коммиту файлы, новые файлы неконтролируемые **git**.
- `git add` - добавляет изменения в рабочей директории в индекс для последующего коммита.
		`git add <file>` — добавляет конкретный файл.
		`git add .` — добавляет все изменения в текущей директории.
		`git add -A` — добавляет все изменения во всем репозитории.
- `git commit` - фиксирует изменения, добавленные в индекс, и создает новый коммит.
		`git commit -m "Сообщение"` — создает коммит с указанным сообщением.
		ОБЯЗАТЕЛЬНО НАДО ПИСАТЬ ОСМЫСЛЕННЫЙ КОММЕНТАРИЙ ТОГО, ЧТО ВЫ СДЕЛАЛИ.
		Коммит также позволяет определить пользователя, который закоммитил говно, по хэшу, который выводится после коммита.
- `git push` - отправляет локальные коммиты в удаленный репозиторий.
		`git push origin main` — отправляет коммиты из ветки `main` в удаленный репозиторий `origin`.
		`git push -u origin <branch>` — при первом push в ветку устанавливает связь с удаленной веткой (флаг `-u`).
- `git branch` - управление ветками.
		`git branch` — показывает список локальных веток.
		`git branch <name>` — создает новую ветку с указанным именем.
		`git branch -d <name>` — удаляет ветку (если она уже слита).
		`git branch -D <name>` — принудительное удаление ветки (даже если не слита).
- `git merge` - сливает указанную ветку с текущей.
		`git merge <branch>` — применяет изменения из указанной ветки к текущей.
		Если возникают конфликты, их нужно разрешить вручную.
- `git checkout` - переключение между ветками, коммитами или файлами.
		`git checkout <branch>` — переключается на указанную ветку.
		`git checkout -b <new-branch>` — создает новую ветку и переключается на нее.
		`git checkout <commit-hash>` — переходит к конкретному коммиту (режим detached HEAD).
- `git pull` - забирает изменения из удаленного репозитория и сливает их с текущей веткой.
